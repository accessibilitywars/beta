Contributor's Guide

AW2 April 12, 2024

The Big Picture

Accessibility Wars uses a static site generator. This means there is no database, no live scripts, and no way to edit directly on the site.

However, it is built from source code and contributors can make edits and submissions using GitHub's interface (or the git tool directly).

How to Contribute

Setup

  • Sign up for an account on GitHub. Feel free to use whatever name you wish. Be aware all your proposed changes will be associated with this name and it will be public.
  • Familiarize yourself with basic markdown

Editing an existing build

  1. Find a build to edit on the source repository. All builds live in this github folder
  2. Click the markdown (.md) file to view it in the github editor
  3. Click the pencil icon to switch to edit mode
  4. Make your changes
  5. Click the Commit Changes... button
  6. Give it a reasonable 1-line commit message and description of what was changed (Berdandy will be reading these so please make them make sense). For example:
  • Commit message: updated mad bomber mechanist to latest balance
  • Extended description: old mad bomber mechanist was missing relics and needed a full trait rework
  1. Ensure your email is correct
  2. Ensure Create a new branch for this commit and start a pull request is selected. Default name is very software oriented (username-patch-N), change it to make sense
  • mad-bomber-balance-update-jan-balance
  1. Click Propose changes
  2. This opens a pull request (git's terminology for a "request for the maintainer to pull your changes in")
  3. Again, title and description. Title should be same as step 9. Add description if you wish
  4. Click Create pull request

From here, the maintainers (Berdandy) will get notified about the submitted changes and can merge them once reviewed. After that, the website builds and should show up in about 2-5 minutes.
